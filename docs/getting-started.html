Getting Started

Grok16 @ 16.1.1 ships a unified g16 driver. One binary compiles C and C++; it picks libexec/grok16/g16-cc or g16-cxx automatically. g++16 is a symlink for compatibility. See Master Coder C and Master Coder C++.

Requirements

  • Linux x86_64
  • Host gcc/g++, git, cmake, python3
  • GCC build deps (gmp, mpfr, mpc, flex, bison, etc.)
  • ~6 GB disk for vendor + build + prefix

First bootstrap

git clone https://github.com/ZacharyGeurts/Grok16.git
cd Grok16
git checkout v16.1.1
export G16_PREFIX="$(pwd)"

./scripts/grok16-toolchain.sh bootstrap
./scripts/grok16-toolchain.sh rebuild
./scripts/grok16-toolchain.sh verify
./scripts/grok16-toolchain.sh field-bench

Rebuild modes

ModeEnvBehavior
Default(none)G16_FAST_REBUILD=1 — incremental, no 3-stage bootstrap
FullG16_FULL_REBUILD=1distclean + 3-stage bootstrap
ReleaseG16_RELEASE_PROFILE=1LTO + PGO + field_opt on forge
./scripts/grok16-toolchain.sh rebuild
G16_FULL_REBUILD=1 ./scripts/grok16-toolchain.sh rebuild
G16_RELEASE_PROFILE=1 ./scripts/grok16-toolchain.sh rebuild

Verify

./scripts/grok16-toolchain.sh verify
./scripts/grok16-toolchain.sh status
./scripts/grok16-toolchain.sh paths

Benchmarks

./scripts/grok16-toolchain.sh field-bench
./scripts/grok16-toolchain.sh bench-all
./scripts/grok16-toolchain.sh profile          # PGO generate → data/pgo/
G16_ENABLE_PGO=1 ./scripts/grok16-toolchain.sh field-bench

CMake consumer

cmake -S examples/minimal-cmake-project -B examples/minimal-cmake-project/build \
  -DCMAKE_TOOLCHAIN_FILE=cmake/grok16-toolchain.cmake
cmake --build examples/minimal-cmake-project/build

Queen consolidate

export GROK16_QUEEN_ROOT=/path/to/NewLatest/Queen   # optional
./scripts/consolidate.sh
./scripts/grok16-toolchain.sh rebuild

Docker

docker build -t grok16 .
docker run --rm grok16 ./scripts/grok16-toolchain.sh status
